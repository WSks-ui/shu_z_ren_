# 《研伴》教育数字人演示视频脚本

## 一、项目技术栈与开发信息

### 1.1 开发平台

| 项目 | 详情 |
|------|------|
| **项目名称** | 研伴 - 智能科研助手 |
| **开发环境** | IntelliJ IDEA |
| **运行环境** | Electron 桌面应用 (跨平台) |
| **目标用户** | 高等教育学生/研究人员 |
| **许可证** | AGPL-3.0 |

### 1.2 前端技术栈

| 技术 | 用途 |
|------|------|
| **Vue.js 3** | 核心框架，Composition API (setup) |
| **Element Plus** | UI 组件库 |
| **Chart.js** | 成长数据可视化 |
| **KaTeX** | 数学公式渲染 |
| **Markdown-it** | Markdown 渲染 |
| **Font Awesome** | 图标库 |
| **Google Fonts** | Outfit + JetBrains Mono 字体 |

### 1.3 数字人技术

| 技术 | 用途 |
|------|------|
| **魔珐星云 SDK** | 3D 数字人形象 + 实时语音合成 |
| **Web Speech API** | 备用语音识别 |
| **流式 TTS** | 句级队列播报，支持打断 |

### 1.4 后端 API

| 接口 | 功能 |
|------|------|
| `/api/education/chat/stream` | 流式对话（SSE） |
| `/api/xingyun_digital_human/config` | 魔珐星云配置 |
| `/api/formula_ocr` | 手写公式识别 |
| `/api/pdf_extract` | PDF 文献解析 |
| `/api/education/achievements` | 成就系统 |
| `/api/education/collaboration` | 协作记录 |

### 1.5 核心功能模块

**五大教育技能**：
```
科研助手 → 选题探索 → 假设构建 → 实验设计 → 数据分析
文献综述 → 主题界定 → 检索策略 → 筛选评估 → 综合撰写
论文写作 → 论文选题 → 框架搭建 → 内容撰写 → 润色定稿
虚拟导师 → 学情诊断 → 知识讲解 → 练习巩固 → 总结提升
数学助手 → 问题理解 → 方法选择 → 逐步求解 → 验证总结
```

**成长系统**：
- 等级/经验值机制
- 12 种成就徽章
- 每日登录奖励 + 连续使用加成

**人机协作记录**：
- 对话历史持久化
- 协作统计可视化
- 导出报告功能

### 1.6 开发流程

```
1. 前端开发
   static/education-digital-human.html  → 页面结构
   static/js/education-app.js           → Vue 应用逻辑 (2700+ 行)
   static/css/education-structure.css   → 样式

2. 后端开发
   py/agent.py, py/skills.py            → Python 后端服务
   提供 RESTful API + SSE 流式响应

3. 集成魔珐星云
   - 申请 AppId/AppSecret
   - 加载 XmovAvatarClass SDK
   - 实现 speak() / interactiveidle() 语音交互

4. 测试运行
   node start.js → 启动 Electron 应用

5. 打包发布
   npm run build:win  → Windows 安装包
   npm run build:mac  → macOS 安装包
   npm run build:linux → Linux 安装包
```

### 1.7 关键技术亮点

1. **流式语音合成** - AI 回复逐句播报，支持打断
2. **LaTeX 过滤** - TTS 前自动过滤数学公式符号
3. **阶段追踪** - AI 可通过 `[阶段:xxx]` 指令切换工作流阶段
4. **多模态输入** - 支持文字、语音、手写公式、PDF 文献

### 【0:45-2:15】 场景一：科研助手——协同式实验设计

**画面**：
- 展示"科研助手"技能模块
- 用户输入研究问题的操作演示

**演示步骤**：

1. **问题输入**
   - 用户输入："我想研究社交媒体对大学生睡眠质量的影响"

2. **苏格拉底式引导**
   - 数字人提问："你的研究假设是什么？"
   - 数字人提问："如何操作化定义'社交媒体使用'这个变量？"
   - 数字人提问："需要控制哪些混淆变量？"

3. **四阶段工作流展示**
   - 选题探索 → 假设构建 → 实验设计 → 数据分析
   - 阶段进度可视化追踪

4. **工具栏功能演示**
   - 点击"定义变量"：梳理自变量、因变量、控制变量
   - 点击"统计方法"：推荐合适的统计分析方法
   - 点击"实验检查"：审视样本量、控制变量、混淆因素
   - 点击"伦理审查"：知情同意、隐私保护、数据安全提示

**旁白**：
> "科研助手采用苏格拉底式引导，而非直接给出答案。它通过提问帮助学习者构建研究假设、定义变量、识别控制变量。这种协同模式培养的是科研思维，而非简单的知识传递。"
>
> "系统内置伦理审查功能，提醒研究者在学术规范框架下开展研究，这正是'以人为本'教育立场的体现。"

**字幕要点**：
- 协同式智能：共同设计实验
- 培养科研思维，而非简单问答
- 内置学术伦理提醒
- 方法论层面的启发性

**评分对应**：
- 技术精准回应研究生科研痛点
- 强调协同式智能（共同设计实验）

5. **协作记录面板**

**字幕要点**：
