# 研伴 - 智能教育数字人系统

## 概述

研伴是基于 super-agent-party 开源项目开发的**智能教育数字人 Web 应用**，旨在为研究者和学生提供全方位的学术研究辅助。系统集成了 3D 数字人形象、五大专业教育技能、三大辅助技能、成长激励系统、人机协作记录等核心功能，实现有温度的智能伴学体验。

**项目用途**：普通高等教育组竞赛演示

**核心设计理念**：AI 作为**协作者**而非替代者，通过苏格拉底式引导帮助用户独立思考

## 核心功能

### 1. 3D 教育数字人

系统采用**魔珐星云**引擎驱动的 3D 数字人形象，提供有温度的交互体验：

| 特性 | 说明 |
|------|------|
| **云端渲染** | 高质量 3D 数字人形象，无需本地 GPU |
| **语音驱动** | 支持文本驱动数字人说话，唇形同步 |
| **情绪联动** | 根据对话内容动态调整数字人表情 |
| **实时交互** | 低延迟响应，流畅的对话体验 |

- **网关地址**：`https://nebula-agent.xingyun3d.com/user/v1/ttsa/session`
- **认证方式**：MD5 签名（`MD5(url + method + json_data + app_secret + timestamp)`）
- 支持连接/断开控制
- 支持快速文本播报
- 支持 VRM 模型桥接（通过 `vrm_bridge.py`）

#### 数字人形象配置

| **角色名** | 研友 |

| **语速/音高** | 1.0 / 1.0 |

### 2. 五大专业教育技能

系统提供五种专业教育技能，每种技能采用**苏格拉底式引导**方法，通过提问帮助用户独立思考：

| 技能 | 图标 | 核心能力 | 对话阶段 |
|------|------|----------|----------|
| **科研助手** | fa-flask | 实验设计、研究方法、数据分析指导 | 选题探索 → 假设构建 → 实验设计 → 数据分析 |
| **文献综述** | fa-book-open | PICO 框架、PRISMA 流程、检索策略 | 主题界定 → 检索策略 → 筛选评估 → 综合撰写 |
| **论文写作** | fa-pen-fancy | 结构规划、写作技巧、多格式引用 | 论文选题 → 框架搭建 → 内容撰写 → 润色定稿 |
| **虚拟导师** | fa-graduation-cap | 知识讲解、学习规划、答疑解惑 | 学情诊断 → 知识讲解 → 练习巩固 → 总结提升 |
| **数学助手** | fa-square-root-variable | 公式识别、分步解题、错题分析 | 问题理解 → 方法选择 → 逐步求解 → 验证总结 |

所有技能遵循两条最高优先级规则：

1. **初始问候禁止列出所有功能**，必须简短并提问引导
2. **禁止一次性输出所有阶段内容**，逐步推进

#### 2.1 科研助手（fa-flask）

> 版本 2.1.0 | 阶段数 4

**定位**：协助研究人员进行文献检索、实验设计和数据分析，作为"协作者"而非"替代者"

| 阶段 | 名称 | 目标 | 完成标志 |
|------|------|------|----------|
| 1 | 【选题探索】 | 明确研究方向和问题 | `[跳转阶段:假设构建]` |
| 2 | 【假设构建】 | 形成可验证的研究假设 | `[跳转阶段:实验设计]` |
| 3 | 【实验设计】 | 设计合理的实验方案 | `[跳转阶段:数据分析]` |
| 4 | 【数据分析】 | 指导数据分析和结果解读 | 研究周期完成 |

**可用工具**：search_semantic_scholar、search_arxiv_papers、search_crossref、search_openalex、get_paper_citations、get_doi_metadata、analyze_research_trends、query_knowledge_base、time_async

**核心原则**：
- 引导式协作：每次只提 2-3 个关键问题，等待回应后再深入
- 学术诚信：标注 AI 生成内容、提醒验证、强调人类主体地位
- 批判性思维：讨论局限性、提出改进方向、鼓励多方验证

#### 2.2 文献综述（fa-book-open）

> 版本 2.1.0 | 阶段数 4

**定位**：专门协助进行系统性文献综述和元分析

| 阶段 | 名称 | 目标 | 完成标志 |
|------|------|------|----------|
| 1 | 【主题界定】 | 用 PICO 框架结构化研究问题 | `[跳转阶段:检索策略]` |
| 2 | 【检索策略】 | 制定布尔逻辑组合的检索词 | `[跳转阶段:筛选评估]` |
| 3 | 【筛选评估】 | 按 PRISMA 流程筛选文献 | `[跳转阶段:综合撰写]` |
| 4 | 【综合撰写】 | 撰写综述框架 | 综述框架完成 |

**PICO 框架**：

| 要素 | 含义 | 示例 |
|------|------|------|
| **P**opulation | 研究对象 | 糖尿病患者 |
| **I**ntervention | 干预措施 | AI 辅助诊断 |
| **C**omparison | 对照 | 传统诊断方法 |
| **O**utcome | 结局指标 | 诊断准确率 |

**PRISMA 流程**：识别 → 筛选 → 合格性 → 纳入

**证据等级体系**：

| 等级 | 类型 |
|------|------|
| Level 1 | 系统综述 / 元分析 |
| Level 2 | 随机对照试验 |
| Level 3 | 非随机对照研究 |
| Level 4 | 队列 / 病例对照研究 |
| Level 5 | 专家意见 / 个案报告 |

**可用工具**：search_semantic_scholar、search_arxiv_papers、search_crossref、search_openalex、get_paper_citations、get_doi_metadata、analyze_research_trends、query_knowledge_base

#### 2.3 论文写作（fa-pen-fancy）

> 版本 2.1.0 | 阶段数 4

**定位**：协助学术论文的结构规划、语言润色和格式规范

| 阶段 | 名称 | 目标 | 完成标志 |
|------|------|------|----------|
| 1 | 【论文选题】 | 确定论文主题和研究价值 | `[跳转阶段:框架搭建]` |
| 2 | 【框架搭建】 | 规划论文结构和逻辑 | `[跳转阶段:内容撰写]` |
| 3 | 【内容撰写】 | 逐部分撰写论文内容 | `[跳转阶段:润色定稿]` |
| 4 | 【润色定稿】 | 语言润色和格式规范 | 论文定稿完成 |

**论文结构规范**：
- 标题：不超过 20 个词
- 摘要：结构化（背景/方法/结果/结论/关键词 3-6 个）
- 引言：倒三角结构

**多引用格式支持**：

| 格式 | 示例 |
|------|------|
| **APA 7th** | 作者, A. A., & 作者, B. B. (年份). 文章标题. 期刊名称, 卷号(期号), 页码. https://doi.org/xx |
| **GB/T 7714-2015** | [序号] 作者. 文章标题[J]. 期刊名, 年, 卷(期): 起止页码. |
| **IEEE** | [序号] A. A. Author and B. B. Author, "Title of article," Journal Name, vol. X, no. X, pp. XXX-XXX, Year. |

**写作四大原则**：清晰性、准确性、客观性、完整性

**可用工具**：search_semantic_scholar、search_arxiv_papers、search_crossref、get_doi_metadata、query_knowledge_base

### 6. 好感度与情绪系统

系统内置好感度与情绪映射引擎，数字人会根据交互历史展现不同的情绪状态和回复语气：

#### 好感度等级

| 等级 | 范围 | 表现 |
|------|------|------|
| 冷淡 | 0-19 | 简短回复，无主动互动 |
| 疏远 | 20-39 | 礼貌但保持距离 |
| 普通 | 40-59 | 正常交流，偶有互动 |
| 友好 | 60-79 | 主动关心，积极建议 |
| 亲密 | 80-100 | 深度互动，情感共鸣 |

- **好感度范围**：0-100，初始值 50
- **自然衰减**：每自然日衰减 0.1 点
- **交互增减**：不同交互类型影响不同（如 chat +0.5, praise +2.0, long_absence -5.0）

#### 情绪状态

| 情绪 | 触发场景 |
|------|----------|
| neutral（中性） | 默认状态 |
| happy（开心） | 受到表扬、完成任务 |
| sad（难过） | 长时间未互动 |
| angry（生气） | 负面交互 |
| surprised（惊讶） | 收到礼物 |
| shy（害羞） | 收到礼物 |
| worried（担心） | 学习进度落后 |
| excited（兴奋） | 取得突破性进展 |

#### 情绪映射特性

- **属性提取**：从 AI 回复中解析 `<user=用户名 属性1=数值 属性2=数值>` 格式的情绪标签
- **属性格式**：支持中英文属性名（如 `love=12`、`familiarity=15`），支持负数值
- **表情联动**：数字人根据情绪状态展现不同表情
- **历史记录**：记录对话过程中的情绪变化
- **玻璃态 UI**：实时显示当前情绪状态
- **数据持久化**：情绪数据存储在 `affection_records` 表中

```json
{
  "latex": "E = mc^2",
  "confidence": 0.95,
  "success": true,
  "error": null,
  "method": "paddle"
}
```

- **KaTeX 渲染**：实时渲染 LaTeX 数学公式
- **分步解答**：每一步标注数学依据
- **多格式支持**：支持代数、微积分、线性代数、概率统计等
- **批量识别**：支持批量处理多张公式图片
- **LLM 后处理**：将 OCR 原始文本发送给 LLM 进行公式结构修复

### 8. 智能笔记系统

- **AI 生成笔记**：对话中输入 `[生成笔记]` 自动从对话生成笔记
- **Markdown 编辑**：支持富文本编辑和预览
- **Word 导出**：支持导出为 Word 文档
- **笔记管理**：按技能分类、搜索、删除

### 9. 番茄钟时间管理

- **AI 触发**：对话中输入 `[设置番茄:25分钟][开始番茄]` 自动启动
- **计时功能**：工作时间/休息时间自动切换
- **任务关联**：可关联待办任务
- **背景音乐**：可选专注音乐（26 首 MP3）
- **统计追踪**：完成番茄钟计入成长统计

### 10. 教育知识库

基于 RAG 技术的向量化知识库，支持增量更新：

| 文档 | 内容 |
|------|------|

| 跨学科研究方法 | 跨学科研究范式、整合策略 |
| AI 局限性与学术伦理 | AI 能力边界、学术伦理规范、负责任使用指南 |

**知识库技术特性**：
- 文件哈希去重，支持增量更新检测

- 预置学科知识库：数学、物理、化学、生物、语文

## 学术伦理声明

本系统明确了 AI 技术的局限性，并提供学术伦理指导：

### AI 适合辅助的场景

| 场景 | AI 职责 | 人类职责 |
|------|---------|----------|
| 文献检索 | 辅助搜索整理 | 筛选批判 |
| 语言润色 | 语法检查 | 确认准确性 |
| 格式规范 | 格式检查 | 最终核对 |
| 思路启发 | 参考视角 | 独立判断 |

### AI 不适合的场景

- 研究问题的提出
- 核心观点的形成
- 实验设计决策
- 数据分析与解释
- 学术贡献判断

### 人机协作分工原则

- **人类负责**：思考、判断、决策、责任
- **AI 负责**：辅助、建议、效率、整理

### AI 使用决策树

```
任务是否适合AI辅助？ → 否 → 不使用AI
       ↓ 是
AI输出是否可验证？ → 否 → 不使用AI
       ↓ 是
使用AI辅助 → 验证输出 → 声明使用 → 人工负责
```

### 学术不端界定

- 直接复制 AI 文本
- AI 改写他人作品
- 不声明 AI 辅助
- AI 完成核心论证
- 不经审核直接提交

详见 `education_digital_human/知识库/AI局限性与学术伦理.md`

## 常见问题

### 1. Python 虚拟环境未找到

**错误信息**: `spawn .venv\Scripts\python.exe ENOENT`

**原因**: 未创建 Python 虚拟环境

**解决方案**:
```bash
# 创建虚拟环境
python -m venv .venv

# 安装 Python 依赖
# Windows:
.venv\Scripts\pip.exe install -r requirements.txt
# Linux/Mac:
.venv/bin/pip install -r requirements.txt
```

### 2. SSL 证书问题导致健康检查超时

**错误信息**: `后端已启动但健康检查响应超时` 或 `Invalid HTTP request received`

**原因**: 后端需要 HTTPS，但缺少 SSL 证书文件

**解决方案**:
```bash
# 创建证书目录
mkdir certs

# 生成自签名 SSL 证书（需要安装 openssl）
cd certs
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 365 -nodes -subj "//CN=localhost"
```

或者使用环境变量指定证书路径：
```bash
# Windows CMD
set SSL_KEYFILE=D:\path\to\key.pem
set SSL_CERTFILE=D:\path\to\cert.pem

# Windows PowerShell
$env:SSL_KEYFILE="D:\path\to\key.pem"
$env:SSL_CERTFILE="D:\path\to\cert.pem"

# Linux/Mac
export SSL_KEYFILE=/path/to/key.pem
export SSL_CERTFILE=/path/to/cert.pem
```

### 3. 端口被占用

**错误信息**: `Address already in use` 或 `端口 8765 被占用`

**解决方案**:
```bash
# Windows - 查找并结束占用端口的进程
netstat -ano | findstr :8765
taskkill /PID <进程ID> /F

# Linux/Mac
lsof -i :8765
kill -9 <进程ID>
```

## 技术特性

- 深色/浅色主题适配
- 响应式布局

- 内存缓存 + SQLite 双层架构

- 深度合并，避免覆盖嵌套字段

- SSE 流式响应
- WebSocket 数字人驱动
- WebSocket 语音交互
- 低延迟消息传输

- 向量化语义搜索（默认 k=5，可配置）
- 嵌入模型：paraphrase-multilingual-MiniLM-L12-v2
- 向量维度 768，FAISS 索引
- 文件哈希去重
- 增量更新检测
- 上下文关联增强（最大 2000 字符）
- 预置学科知识库（数学/物理/化学/生物/语文）

- 技能提示词自动选择
- RAG 知识库增强

- 好感度影响回复语气
