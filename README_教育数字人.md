# 研伴 - 智能教育数字人系统

## 概述

研伴是基于 super-agent-party 开源项目开发的**智能教育数字人 Web 应用**，旨在为研究者和学生提供全方位的学术研究辅助。系统集成了 3D 数字人形象、五大专业教育技能、成长激励系统、人机协作记录等核心功能，实现有温度的智能伴学体验。

**项目用途**：普通高等教育组竞赛演示

**核心设计理念**：AI 作为**协作者**而非替代者，通过苏格拉底式引导帮助用户独立思考

## 核心功能

### 1. 3D 教育数字人

系统采用**魔珐星云**引擎驱动的 3D 数字人形象，提供有温度的交互体验：

| 特性 | 说明 |
|------|------|
| **云端渲染** | 高质量 3D 数字人形象，无需本地 GPU |
| **语音驱动** | 支持文本驱动数字人说话，唇形同步 |
| **情绪联动** | 根据对话内容动态调整数字人表情 |
| **实时交互** | 低延迟响应，流畅的对话体验 |

- **网关地址**：`https://nebula-agent.xingyun3d.com/user/v1/ttsa/session`
- **认证方式**：MD5 签名（`MD5(url + method + json_data + app_secret + timestamp)`）
- 支持连接/断开控制
- 支持快速文本播报

#### 数字人形象配置

### 2. 五大专业教育技能

系统提供五种专业教育技能，每种技能采用**苏格拉底式引导**方法，通过提问帮助用户独立思考：

| 技能 | 图标 | 核心能力 | 对话阶段 |
|------|------|----------|----------|
| **科研助手** | fa-flask | 实验设计、研究方法、数据分析指导 | 选题探索 → 假设构建 → 实验设计 → 数据分析 |
| **文献综述** | fa-book-open | PICO 框架、PRISMA 流程、检索策略 | 主题界定 → 检索策略 → 筛选评估 → 综合撰写 |
| **论文写作** | fa-pen-fancy | 结构规划、写作技巧、多格式引用 | 论文选题 → 框架搭建 → 内容撰写 → 润色定稿 |
| **虚拟导师** | fa-graduation-cap | 知识讲解、学习规划、答疑解惑 | 学情诊断 → 知识讲解 → 练习巩固 → 总结提升 |
| **数学助手** | fa-square-root-variable | 公式识别、分步解题、错题分析 | 问题理解 → 方法选择 → 逐步求解 → 验证总结 |

所有技能遵循两条最高优先级规则：

1. **初始问候禁止列出所有功能**，必须简短并提问引导
2. **禁止一次性输出所有阶段内容**，逐步推进

#### 2.1 科研助手（fa-flask）

> 版本 2.1.0 | 阶段数 4

**定位**：协助研究人员进行文献检索、实验设计和数据分析，作为"协作者"而非"替代者"

| 阶段 | 名称 | 目标 | 完成标志 |
|------|------|------|----------|
| 1 | 【选题探索】 | 明确研究方向和问题 | `[跳转阶段:假设构建]` |
| 2 | 【假设构建】 | 形成可验证的研究假设 | `[跳转阶段:实验设计]` |
| 3 | 【实验设计】 | 设计合理的实验方案 | `[跳转阶段:数据分析]` |
| 4 | 【数据分析】 | 指导数据分析和结果解读 | 研究周期完成 |

**可用工具**：search_semantic_scholar、search_arxiv_papers、search_crossref、search_openalex、get_paper_citations、get_doi_metadata、analyze_research_trends、query_knowledge_base、time_async

**核心原则**：
- 引导式协作：每次只提 2-3 个关键问题，等待回应后再深入
- 学术诚信：标注 AI 生成内容、提醒验证、强调人类主体地位
- 批判性思维：讨论局限性、提出改进方向、鼓励多方验证

#### 2.2 文献综述（fa-book-open）

> 版本 2.1.0 | 阶段数 4

**定位**：专门协助进行系统性文献综述和元分析

| 阶段 | 名称 | 目标 | 完成标志 |
|------|------|------|----------|
| 1 | 【主题界定】 | 用 PICO 框架结构化研究问题 | `[跳转阶段:检索策略]` |
| 2 | 【检索策略】 | 制定布尔逻辑组合的检索词 | `[跳转阶段:筛选评估]` |
| 3 | 【筛选评估】 | 按 PRISMA 流程筛选文献 | `[跳转阶段:综合撰写]` |
| 4 | 【综合撰写】 | 撰写综述框架 | 综述框架完成 |

**PICO 框架**：

| 要素 | 含义 | 示例 |
|------|------|------|
| **P**opulation | 研究对象 | 糖尿病患者 |
| **I**ntervention | 干预措施 | AI 辅助诊断 |
| **C**omparison | 对照 | 传统诊断方法 |
| **O**utcome | 结局指标 | 诊断准确率 |

**PRISMA 流程**：识别 → 筛选 → 合格性 → 纳入

**证据等级体系**：

| 等级 | 类型 |
|------|------|
| Level 1 | 系统综述 / 元分析 |
| Level 2 | 随机对照试验 |
| Level 3 | 非随机对照研究 |
| Level 4 | 队列 / 病例对照研究 |
| Level 5 | 专家意见 / 个案报告 |

**可用工具**：search_semantic_scholar、search_arxiv_papers、search_crossref、search_openalex、get_paper_citations、get_doi_metadata、analyze_research_trends、query_knowledge_base

#### 2.3 论文写作（fa-pen-fancy）

> 版本 2.1.0 | 阶段数 4

**定位**：协助学术论文的结构规划、语言润色和格式规范

| 阶段 | 名称 | 目标 | 完成标志 |
|------|------|------|----------|
| 1 | 【论文选题】 | 确定论文主题和研究价值 | `[跳转阶段:框架搭建]` |
| 2 | 【框架搭建】 | 规划论文结构和逻辑 | `[跳转阶段:内容撰写]` |
| 3 | 【内容撰写】 | 逐部分撰写论文内容 | `[跳转阶段:润色定稿]` |
| 4 | 【润色定稿】 | 语言润色和格式规范 | 论文定稿完成 |

**论文结构规范**：
- 标题：不超过 20 个词
- 摘要：结构化（背景/方法/结果/结论/关键词 3-6 个）
- 引言：倒三角结构

**多引用格式支持**：

| 格式 | 示例 |
|------|------|
| **APA 7th** | 作者, A. A., & 作者, B. B. (年份). 文章标题. 期刊名称, 卷号(期号), 页码. https://doi.org/xx |
| **GB/T 7714-2015** | [序号] 作者. 文章标题[J]. 期刊名, 年, 卷(期): 起止页码. |
| **IEEE** | [序号] A. A. Author and B. B. Author, "Title of article," Journal Name, vol. X, no. X, pp. XXX-XXX, Year. |

**写作四大原则**：清晰性、准确性、客观性、完整性

**可用工具**：search_semantic_scholar、search_arxiv_papers、search_crossref、get_doi_metadata、query_knowledge_base

#### 2.4 虚拟导师（fa-graduation-cap）

> 版本 2.0.0 | 阶段数 4

**定位**：作为虚拟导师进行个性化教学和知识讲解

| 阶段 | 名称 | 目标 | 完成标志 |
|------|------|------|----------|
| 1 | 【学情诊断】 | 了解学习者知识水平和需求 | `[跳转阶段:知识讲解]` |
| 2 | 【知识讲解】 | 针对性讲解核心概念 | `[跳转阶段:练习巩固]` |
| 3 | 【练习巩固】 | 通过练习加深理解 | `[跳转阶段:总结提升]` |
| 4 | 【总结提升】 | 总结要点并拓展提升 | 学习周期完成 |

**三大教学理念**：
- **以学习者为中心**：关注需求、尊重节奏、鼓励主动参与
- **建构主义学习**：激活已有知识 → 构建知识联系 → 促进意义建构
- **支架式教学**：提供适度支持 → 逐步撤除支架 → 培养独立能力

**教学策略**：苏格拉底式提问（4 个层次）、类比教学、分步讲解

**学科覆盖**：

| 领域 | 内容 |
|------|------|
| 自然科学 | 概念理解、公式推导、实验设计 |
| 社会科学 | 理论框架、研究方法、案例分析 |
| 工程技术 | 原理解释、问题排查、设计思路 |
| 人文学科 | 文本解读、观点分析、论证结构 |

**可用工具**：search_semantic_scholar、get_wikipedia_summary_and_sections、get_wikipedia_section_content、query_knowledge_base、time_async

#### 2.5 数学助手（fa-square-root-variable）

> 版本 1.0.0 | 阶段数 4

**定位**：手写公式识别、数学推导与解题辅助

| 阶段 | 名称 | 目标 | 完成标志 |
|------|------|------|----------|
| 1 | 【问题理解】 | 理解数学问题的条件和目标 | `[跳转阶段:方法选择]` |
| 2 | 【方法选择】 | 选择合适的解题方法 | `[跳转阶段:逐步求解]` |
| 3 | 【逐步求解】 | 分步骤推导求解 | `[跳转阶段:验证总结]` |
| 4 | 【验证总结】 | 验证结果并总结方法 | 解题完成 |

**数学领域覆盖**：

| 领域 | 内容 |
|------|------|
| 基础数学 | 四则运算与方程、函数与图像、几何与三角 |
| 高等数学 | 微积分（极限/导数/积分）、线性代数（矩阵/向量/特征值）、概率统计 |
| 离散数学 | 集合论、逻辑与证明、图论 |
| 应用数学 | 数值计算、优化问题、数学建模 |

**LaTeX 公式支持**：

| 符号 | LaTeX 写法 |
|------|-----------|
| 分数 | `\frac{a}{b}` |
| 上下标 | `x^2` / `a_n` |
| 根号 | `\sqrt{x}` / `\sqrt[n]{x}` |
| 求和 | `\sum_{i=1}^{n}` |
| 积分 | `\int_{a}^{b}` |
| 矩阵 | `\begin{pmatrix} a & b \\ c & d \end{pmatrix}` |

**手写公式识别流程**：点击"手写公式"按钮 → 画板书写 → 系统识别转为 LaTeX → 进一步分析计算

**错题分析五步法**：逐步检查 → 精确定位 → 分析原因 → 正确解法 → 总结易错点

**可用工具**：query_knowledge_base、time_async

#### 技能通用特性

- 支持图片识别（公式、文献、笔记、图表等）
- 支持阶段跳转指令 `[跳转阶段:阶段名]`
- 支持番茄钟时间管理 `[设置番茄:25分钟][开始番茄]`
- 支持笔记生成 `[生成笔记]`
- 支持多轮对话和长期记忆
- 每个技能独立追踪使用次数
- 区分用户手动跳转和 AI 触发跳转，AI 向前推进阶段时触发经验奖励

### 3. 成长与成就系统

#### 经验与等级

| 功能 | 说明 |
|------|------|
| **经验值** | 通过对话、使用技能、解锁成就获得经验值 |
| **等级系统** | 每 100 经验升一级，支持多级成长 |
| **技能统计** | 分别统计各技能使用次数 |
| **综合统计** | 对话次数、论文阅读、实验设计、学习时长等 |

| 分类 | 成就 | 解锁条件 | 奖励经验 |
|------|------|----------|----------|

| | 技能探索者 | 使用所有技能各1次 | +50 |
| **进阶成就** | 进阶学者 | 达到5级 | +100 |
| | 协作大师 | 完成20次协作会话 | +80 |
| | 番茄大师 | 完成10个番茄钟 | +60 |
| | 知识收藏家 | 收藏20条知识书签 | +70 |
| **挑战成就** | 资深学者 | 达到10级 | +200 |
| | 任务达人 | 完成50个待办任务 | +150 |
| | 公式达人 | 识别10个手写公式 | +100 |
| | 阶段大师 | 完成20个学习阶段 | +120 |
| **隐藏成就** | 夜间学者 | 在深夜(23:00-5:00)使用研伴 | +50 |

**成就系统特性**：
- 点击徽章查看详细信息（描述、分类、奖励、解锁时间）

- 解锁时自动发放经验奖励
- 成就进度实时追踪

### 4. 人机协作记录

- **协作类型**：论文协作、实验协作、综述协作、辅导协作
- **贡献追踪**：分别记录 AI 和用户的贡献内容
- **协作比例**：自动计算人机协作贡献比例
- **报告导出**：支持导出 Word 格式学习报告
- **选择性导出**：支持按类型筛选、指定会话导出

#### 学习报告功能

学习报告包含以下内容：

| 模块 | 内容 |
|------|------|
| **核心统计** | 学习会话数、总贡献、对话次数、当前等级 |
| **成就时间线** | 最近解锁的成就及解锁日期 |
| **技能使用趋势** | 各技能使用次数条形图、最常用技能标注 |
| **智能学习建议** | 优势领域、改进建议、下一步目标、学习技巧 |

### 5. 情绪映射引擎

- **情绪提取**：从 AI 回复中解析 `<user=用户名 属性1=数值 属性2=数值>` 格式的情绪标签
- **属性格式**：支持中英文属性名（如 `love=12`、`familiarity=15`），支持负数值
- **表情联动**：数字人根据情绪状态展现不同表情
- **历史记录**：记录对话过程中的情绪变化
- **玻璃态 UI**：实时显示当前情绪状态
- **数据持久化**：情绪数据存储在 `affection/affection_data.json`

### 6. 数学公式支持

#### 双引擎识别

| 引擎 | 说明 | 置信度 |
|------|------|--------|
| **SimpleTex**（主方案） | 标准/加速两种模式，API Key 从环境变量 `SIMPLETEX_API_KEY` 获取 | 原始置信度 |
| **多模态 LLM**（回退方案） | 使用 GPT-4o 等多模态模型识别，SimpleTex 失败时自动切换 | 固定 0.85 |

**识别返回格式**：
```json
{
  "latex": "E = mc^2",
  "confidence": 0.95,
  "success": true,
  "error": null,
  "method": "simpletex"
}
```

- **KaTeX 渲染**：实时渲染 LaTeX 数学公式
- **分步解答**：每一步标注数学依据
- **多格式支持**：支持代数、微积分、线性代数、概率统计等

### 9. 教育知识库

基于 RAG 技术的向量化知识库，支持增量更新：

| 文档 | 内容 |
|------|------|
| 学术写作规范 | 论文结构、语言风格、引用格式指南 |
| 教育研究方法 | 定量/定性研究设计、数据分析方法 |
| 跨学科研究方法 | 跨学科研究范式、整合策略 |
| AI 局限性与学术伦理 | AI 能力边界、学术伦理规范、负责任使用指南 |

**知识库技术特性**：
- 文件哈希去重，支持增量更新检测
- 向量化语义搜索（默认 k=3）
- 上下文关联增强（最大长度 2000 字符）
- 预加载 + 懒加载双模式

### 10. 学术 API 集成

## 数据存储架构

### 双层存储设计

```
┌─────────────────────────────────────┐
│         内存缓存 (MemoryCache)       │
│  • dict 存储，低延迟读写             │
│  • 脏数据追踪 (_dirty)              │
│  • 5秒批量刷新到 SQLite             │
│  • 深度合并，避免覆盖嵌套字段        │
└──────────────┬──────────────────────┘
               ↓ 每5秒 / 立即写入
┌─────────────────────────────────────┐
│         SQLite 持久化               │
│  • education_data.db                │
│  • 5张表 + 2个索引                  │
│  • 自动迁移旧 JSON 数据             │
└─────────────────────────────────────┘
```

## 学术伦理声明

本系统明确了 AI 技术的局限性，并提供学术伦理指导：

### AI 适合辅助的场景

| 场景 | AI 职责 | 人类职责 |
|------|---------|----------|
| 文献检索 | 辅助搜索整理 | 筛选批判 |
| 语言润色 | 语法检查 | 确认准确性 |
| 格式规范 | 格式检查 | 最终核对 |
| 思路启发 | 参考视角 | 独立判断 |

### AI 不适合的场景

- 研究问题的提出
- 核心观点的形成
- 实验设计决策
- 数据分析与解释
- 学术贡献判断

### 人机协作分工原则

- **人类负责**：思考、判断、决策、责任
- **AI 负责**：辅助、建议、效率、整理

### AI 使用决策树

```
任务是否适合AI辅助？ → 否 → 不使用AI
       ↓ 是
AI输出是否可验证？ → 否 → 不使用AI
       ↓ 是
使用AI辅助 → 验证输出 → 声明使用 → 人工负责
```

### 学术不端界定

- 直接复制 AI 文本
- AI 改写他人作品
- 不声明 AI 辅助
- AI 完成核心论证
- 不经审核直接提交

详见 `education_digital_human/知识库/AI局限性与学术伦理.md`

## 常见问题

### 1. Python 虚拟环境未找到

**错误信息**: `spawn .venv\Scripts\python.exe ENOENT`

**原因**: 未创建 Python 虚拟环境

**解决方案**:
```bash
# 创建虚拟环境
python -m venv .venv

# 安装 Python 依赖
# Windows:
.venv\Scripts\pip.exe install -r requirements.txt
# Linux/Mac:
.venv/bin/pip install -r requirements.txt
```

### 2. SSL 证书问题导致健康检查超时

**错误信息**: `后端已启动但健康检查响应超时` 或 `Invalid HTTP request received`

**原因**: 后端需要 HTTPS，但缺少 SSL 证书文件

**解决方案**:
```bash
# 创建证书目录
mkdir certs

# 生成自签名 SSL 证书（需要安装 openssl）
cd certs
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 365 -nodes -subj "//CN=localhost"
```

或者使用环境变量指定证书路径：
```bash
# Windows CMD
set SSL_KEYFILE=D:\path\to\key.pem
set SSL_CERTFILE=D:\path\to\cert.pem

# Windows PowerShell
$env:SSL_KEYFILE="D:\path\to\key.pem"
$env:SSL_CERTFILE="D:\path\to\cert.pem"

# Linux/Mac
export SSL_KEYFILE=/path/to/key.pem
export SSL_CERTFILE=/path/to/cert.pem
```

### 3. Electron 下载超时（国内网络）

**错误信息**: `RequestError: connect ETIMEDOUT` 或 `npm install` 卡住

**原因**: Electron 二进制文件下载超时

**解决方案**:
```bash
# 使用国内镜像
# Windows CMD
set ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/
npm install

# Windows PowerShell
$env:ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"
npm install

# Linux/Mac
ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/ npm install
```

### 4. node_modules 目录被锁定

**错误信息**: `EBUSY: resource busy or locked`

**原因**: 有进程占用了 node_modules 目录

**解决方案**:
1. 关闭所有可能占用的程序（VS Code、其他终端、资源管理器等）
2. 重新打开终端，删除 node_modules 后重新安装：
```bash
# Windows
rmdir /s /q node_modules
npm install

# Linux/Mac
rm -rf node_modules
npm install
```

### 5. 端口被占用

**错误信息**: `Address already in use` 或 `端口 3456 被占用`

**解决方案**:
```bash
# Windows - 查找并结束占用端口的进程
netstat -ano | findstr :3456
taskkill /PID <进程ID> /F

# Linux/Mac
lsof -i :3456
kill -9 <进程ID>
```
